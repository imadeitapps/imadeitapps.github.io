# iMadeIt Apps — Portfolio Website

Source code for [imadeitapps.github.io](https://imadeitapps.github.io).

A static Astro site that automatically displays your App Store catalogue.
**Adding a new app requires editing one file.** Everything else — metadata,
icons, ratings — is fetched from Apple's API at build time.

---

## Quick Start

```bash
# 1. Install dependencies
npm install

# 2. Fetch app metadata from Apple's API
npm run fetch-apps

# 3. Start the dev server
npm run dev          # → http://localhost:4321

# 4. Production build (fetch + build)
npm run build
```

---

## Adding a New App

Open `data/apps.json` and add an entry:

```json
[
  {
    "url": "https://apps.apple.com/app/id6756003398",
    "featured": true
  },
  {
    "url": "https://apps.apple.com/app/id123456789"
  }
]
```

| Field      | Required | Description |
|------------|----------|-------------|
| `url`      | ✅       | Any valid App Store URL containing the numeric ID |
| `featured` | No       | `true` to show in the Featured section above the grid |
| `order`    | No       | Integer to control display order within each group |

That's it. Commit `apps.json` and GitHub Actions builds and deploys the site
automatically. No other file needs to change.

## How Metadata Retrieval Works

`scripts/fetch-apps.js` runs once at build time:

1. Reads `data/apps.json`
2. Extracts the numeric App Store ID from each URL (the `/idXXXXXXX` segment)
3. Calls `https://itunes.apple.com/lookup?id=XXXXXX&country=us` for each ID
4. Extracts: name, subtitle, description, icon (512 px), category, rating,
   rating count, developer name, canonical URL, price, version, and more
5. Writes the combined data to `src/data/apps-metadata.json`
6. Astro imports this static JSON file at build time — no runtime API calls

**Why build-time?** Visitors receive a fully rendered, static page.
No client-side fetches, no CORS issues, no Apple API rate-limiting on load,
and better SEO because app content is present in the HTML.

## Error Handling

The fetch script handles errors gracefully:

| Situation | Behaviour |
|-----------|-----------|
| Invalid URL format | Skipped with a warning; build continues |
| App ID not found in iTunes | Skipped with a warning; build continues |
| Network timeout (12 s) | Skipped with a warning; build continues |
| HTTP error from iTunes API | Skipped with a warning; build continues |
| Zero apps fetched successfully | Script exits with code 1, failing the build |
| Partial success | Build succeeds; failed apps are reported in the log |

A 650 ms pause between requests prevents rate-limiting by Apple's servers.

## GitHub Pages Deployment

### One-time Setup

1. Push this repository to GitHub under the account `imadeitapps`
   (repository name: `imadeitapps.github.io`)

2. Go to **Settings → Pages**

3. Under **Source**, select **GitHub Actions**

4. That's it — the workflow runs automatically on every push to `main`

### Manual Trigger

To re-fetch app metadata without pushing code:

1. Go to **Actions → Build & Deploy to GitHub Pages**
2. Click **Run workflow**

## Local Development

```bash
# Install
npm install

# Populate metadata (required once before npm run dev)
npm run fetch-apps

# Dev server with live reload
npm run dev

# Production build (fetches fresh metadata)
npm run build

# Preview the production build locally
npm run preview

# Build without re-fetching (uses cached apps-metadata.json)
npm run build:static
```

## Customisation

### Editing the Privacy Policy

Open `src/pages/privacy.astro` and edit the HTML directly.
It uses the same navigation and footer as the rest of the site.
Update the `lastUpdated` variable and `contactEmail` at the top of the file.

### Changing the Tagline / Hero Copy

Open `src/pages/index.astro`. The hero section is at the top of the template.

### Design Tokens

All colours, spacing, typography, and shadows are defined as CSS custom
properties in `src/styles/global.css` under the `:root` selector.
Dark mode overrides are in the `@media (prefers-color-scheme: dark)` block
immediately below.

## Maintenance

- **Adding an app:** Edit `data/apps.json`, commit, and push.
- **Removing an app:** Delete its entry from `data/apps.json`.
- **Refreshing ratings/metadata:** Trigger a manual workflow run, or just push
  any change to `main`.
- **Updating the site design:** Edit files in `src/`. No rebuild of metadata needed.
- **Node/Astro updates:** Run `npm update` and test locally before pushing.
